## 本プロジェクトの位置づけ

Giveryで2023年12月から2026年4月まで約80案件の法人向け生成AI研修を設計・実施してきた。
対象はChatGPT入門からRAG構築、AI駆動開発、Difyワークフロー、AIセキュリティまで多岐にわたる。
この蓄積をUdemyのオンデマンド教材として再構成し、日本語圏で圧倒的な生成AI研修シリーズを作る。

法人研修では1社あたり3時間から3日間、対面で濃密にやる。
Udemyではその密度を保ちつつ、1レクチャー5-10分の粒度に分解し、独学で完走できる設計にする。

## Udemy市場調査のサマリ

### 日本語圏の現状
- 生成AI入門系が乱立。評価は4.0-4.2に集中。差別化が難しい
- エンジニア向け深い技術講座は手薄。英語コースに流れている
- 「コード不要」「ビジネス活用」がキーワードとして強い
- Cursor/Vibe Coding系は出始めたばかりで先行者利益がある

### 英語圏のベストセラー
- Ed Donner「AI Engineer Core Track」が4.7/224,632受講者でトップ。8週間8プロジェクトのブートキャンプ型
- 「LLM Engineering」「RAG」「Agents」がトレンドワード
- 86時間超のコースは完走率が低く、15-35時間が最適ボリューム
- ローカルLLM特化（Ollama/LM Studio）が4.8の最高評価

### 高評価コースの共通パターン
- 1レクチャー5-15分、1セクション30-90分
- セクション末にクイズまたはハンズオン
- エンジニア向けはハンズオン比率40-60%
- タイトルに年号+具体的技術名
- 講師の信頼性（実務経験）が購買決定要因

## Udemyコースシリーズ設計

### 設計原則

1. Givery研修の実績をベースに、法人研修で磨いた「型」をそのまま活かす
2. 日本語圏で手薄な「エンジニア向け実践」を主軸に据える
3. 基礎コースには必ずセキュリティ/ガバナンスとトレンドキャッチアップを含める
4. ツール別コースは「基礎」と「応用」を分けて、入口の心理的ハードルを下げる
5. 各コース8-15時間、5-10セクション構成
6. 全コースにハンズオンプロジェクトを含む（完成物を持ち帰れる設計）

### Layer 0: 共通基盤

#### Course 01: 生成AI完全入門 2026

全コースの入口。他の全コースが前提とする土台を作る。
セキュリティ/ガバナンス基礎と生成AIトレンドキャッチアップをこの段階から組み込むことで、
ツール操作を学ぶ前に「使う側の責任」と「技術の現在地」を理解させる。

- 対象: 全レベル（非エンジニア含む）
- 想定時間: 10時間
- 原材料: TIS新入社員研修、SSE0324セキュリティ研修、朝日新聞AIリテラシー、各社トレンド研修
- 構成:
  - Sec1: 生成AIの仕組みと限界（LLMの原理、Transformer、トークン、ハルシネーション）
  - Sec2: 生成AIトレンドキャッチアップ 2026（モデル進化の歴史、GPT/Claude/Gemini/LLaMA系譜、マルチモーダル、エージェント、ローカルLLM、オープンソースの動向）
  - Sec3: 主要ツール比較と選び方（ChatGPT, Claude, Gemini, Copilot, Codex。料金体系、得意不得意、企業導入時の判断基準）
  - Sec4: プロンプトの基本（指示の型3つ、Few-shot、Chain of Thought、構造化出力）
  - Sec5: ビジネス活用パターン（文書作成、要約、分析、翻訳、コードレビュー）
  - Sec6: セキュリティ/ガバナンス基礎（ハルシネーション対策、権利侵害、情報漏えい、プロンプトインジェクションの基本、社内ルール設計の出発点）
  - Sec7: 生成AI資格ロードマップ（G検定、生成AIパスポート、AWS AI Practitioner、Google Cloud AI等）
  - Sec8: ミニプロジェクト（業務改善提案書をAIで作成+リスク評価シート付き）
- 差別化: セキュリティ+トレンドを入門段階で扱う。「使い方」の前に「使う覚悟」を作る

### Layer 1: ツール基礎

各基礎コースには共通で以下を含める:
- そのツール固有のセキュリティ/ガバナンス注意点（データの扱い、料金、利用規約）
- トレンドキャッチアップ（そのツールの最新アップデートと今後の方向性）

#### Course 02: Claude基礎 claude.ai完全ガイド

Anthropicが提供するClaude（claude.ai）の全機能を体系的に学ぶ。
ChatGPTとの違い、Claudeならではの強みを理解し、業務で使いこなす。

- 対象: 初級（ビジネスユーザー/エンジニア）
- 想定時間: 8時間
- 原材料: RgenAI講座素材、各社Claude活用実績、Anthropic公式ドキュメント
- 構成:
  - Sec1: Claudeとは（Anthropicの思想、Constitutional AI、他LLMとの違い）
  - Sec2: claude.ai基本操作（会話、ファイル添付、画像認識、PDF分析）
  - Sec3: Claude Projects（プロジェクト作成、カスタム指示、ナレッジベース構築）
  - Sec4: Claude Artifacts（コード生成、ドキュメント作成、インタラクティブ成果物）
  - Sec5: 長文処理と分析（200Kトークン活用、書籍丸ごと要約、コードベース分析）
  - Sec6: Claude API入門（APIキー取得、Python/TypeScript SDK、基本的なAPI呼び出し）
  - Sec7: セキュリティとデータの扱い（Claudeのデータポリシー、Enterprise vs Pro、情報漏えい防止）
  - Sec8: トレンド（Claude 4.x系の進化、Opus/Sonnet/Haiku使い分け、今後のロードマップ）
  - プロジェクト: 業務特化Claude Projectを3つ構築
- 差別化: claude.aiに特化した日本語コースがほぼ存在しない。API入門まで含めて橋渡し

#### Course 03: Cowork基礎 AIとの協働術

特定のツールに依存しない「AIとの協働（Cowork）」の原則を学ぶ。
どのAIツールを使う場合でも共通する、人間とAIの最適な役割分担の設計法。

- 対象: 初級-中級（全職種）
- 想定時間: 8時間
- 原材料: OD2 Copilot心理的ハードル低減研修、TIS「学習パートナーとしてのAI」、EPSN「AIは代替ではなく補助ツール」
- 構成:
  - Sec1: AI協働の大原則（AIは代替ではなく思考の補助。人間がレビューする前提を崩さない）
  - Sec2: タスク分解の技術（業務をAI向きタスクと人間向きタスクに分ける。業務プロセス分解シート）
  - Sec3: AIへの委譲と品質管理（何を任せ、何を自分でやるか。出力の検証フレームワーク）
  - Sec4: チームでのAI活用（共有プロンプト、ナレッジ管理、スキル格差への対処）
  - Sec5: AI活用の心理的障壁を越える（「難しい」「怖い」「仕事を奪われる」への対処法。OD2研修で実証済み）
  - Sec6: 業務別AI協働パターン（企画、営業、マーケ、人事、経理、エンジニアリング）
  - Sec7: セキュリティとガバナンス（チーム利用時のリスク管理、AIガイドライン運用の実際）
  - Sec8: トレンド（Agentic Workflowの進化が協働をどう変えるか、2026年以降の展望）
  - プロジェクト: 自部門のAI活用計画書（タスク分解+ツール選定+リスク評価+KPI設定）
- 差別化: ツール操作ではなく「協働の設計」を教える。これ単体で受講する価値がある

#### Course 05: GitHub Copilot基礎

- 対象: 初級-中級（エンジニア）
- 想定時間: 8時間
- 原材料: Honda バイブコーディング研修（Codex選定候補）、EPSN研修、Codex公式ドキュメント
